## open-knm 产品需求（中文版）

### 背景与定位
- `open-knm` 是一个围绕荷兰 KNM（融入考试/社会融入）框架、面向新移民的中英双语知识库，目标帮助已经在荷兰或即将来到的在京非母语者快速理解荷兰社会制度、公共服务及日常生活操作，强调「融合导向」而非单独讲 KNM 主题。
- 受众：新移民、留学生、在荷兰工作的新人，需要在中文语境下获取制度、生活与工作信息，同时可以参考英文原文辅助理解或复习。

### 核心目标
1. **KNM 贯穿（不是额外栏目）**  
   - 所有页面（首页、社会/生活/工作学习/资源/文章等）都要体现 KNM 的结构思维，不再单独保留 “KNM” 选项，首页和文章说明中强调「全站内容即 KNM 融入主题的分支」。
2. **双语流程顺畅**  
   - 全局支持 `zh` 与 `en` 语言，导航、摘要、文章标题/描述/内容均优先提供当前语言，缺失时明确提示并提供可用语言的切换链接。
3. **聚焦实践与可操作内容**  
   - 每个主题页展示实用贴士（如租房、签证、医疗、工作合同、税务），同时抛出关键术语与标准表达、相关机构链接，帮助用户在生活中快速执行。
4. **开放协作与反馈**  
   - 文章末尾展示「如何改进」段落 + GH 仓库链接（占位），说明内容来自社区，可提交修改建议；中新语言版本也鼓励贡献。

### 功能需求
- **统一元数据驱动**  
  `src/lib/articles.ts` 维护每篇文章的 slug、分类、中文/英文标题与摘要并导出分类筛选函数，便于主题页、导航与详情页统一读取。
- **MDX 内容与语言加载**  
  `src/app/articles/[slug]/page.tsx` 按 locale 动态 `import("../../../../content/{locale}/{slug}.mdx")`，渲染整洁的 `prose` 样式；若某语言 MDX 缺失，显示提示框并提供跳转到另一语言版本的链接。
- **导航与布局**  
  `src/app/layout.tsx` 包含带 Logo 的导航条、`LanguageSwitcher`、网格化主内容区和 footer；导航项包括 `社会`、`生活`、`工作与学习`、`资源`、`关于`，语言切换按钮在桌面/移动都可见。
- **首页叙事**  
  首页 hero +卡片要突出「开放、实践、KNM 融合」，提供 CTA（如“从社会主题开始”、“查看生活指南”），并通过小节点说明：社会基础、生活实操、KNM 知识、协作开放。
- **主题与资源整理**  
  每个主题页列出相关 article 列表与 brief 介绍，提供 category-specific CTA 连接，帮助用户系统浏览（如 `life`, `society`, `work-study`）。
- **开放资源与反馈入口**  
  Article 页底添加反馈段落说明「内容由社区维护」，并嵌入 GitHub 链接或未来表单入口，提升透明度与贡献导向。

### 技术与架构要求
- 核心框架：Next.js 16 + App Router + React 19 + TypeScript 5。
- 样式栈：Tailwind CSS v4，`src/app/globals.css` 定义荷兰调色板（橙、蓝、柔和背景），并启用 `prose` 相关样式。
- 字体：使用 `next/font` 引入 Geist/Geist Mono，在 body 上挂载 CSS 变量。
- CI/构建：保留标准 `npm run dev/build/start/lint`，确保 `npm run build` 无报错并可部署到类似 Vercel 的平台。
- 内容：`content/{zh,en}` 每篇 MDX 文件按 slug 命名，方便 loader 动态按语言加载。

### 验收条件
- 新建首页、分类页、文章页在两种语言下均可访问；语言切换按钮始终在可见位置。
- 所有文章由 `articles.ts` 提供元数据，Detail 页可加载对应语言 MDX 或提供 fallback 提示。
- 页面导航聚焦 `社会/生活/工作与学习/资源/关于`，并在文案（如首页 hero、footer）中强调「全站即 KNM 融入资源」。
- `docs/prd.md` 内容保持可复制/参考，包含需求背景、目标、功能、技术、验收五个部分。
